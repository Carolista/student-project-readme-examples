<a name="basics-beyond"></a>
### üìö The Basics & Beyond
Markdown is *extremely* easy to learn, and well worth the minor effort it takes to use it if you want your README to be easy-to-read and show off your project well.

Headings
```markdown
# Heading 1
## Heading 2
<!-- etc -->
```

Character Styling
```markdown
Regular
*Italic*
**Bold**
```

Unordered Lists
```markdown
Some things, in no particular order:
- Thing
- Another thing
- Yet another thing
```

Ordered Lists
```markdown
Step-by-step instructions:
1. First, do the thing to get started.
1. Then, do another thing.
1. Finally, do the last thing.
```

Tables
```markdown
| Column 1 | Column 2 | Column 3 |
| :---     |   :--:   |     ---: |
| Left     |  Center  |    Right |
```

... and so much more! The resources below will teach you quite a lot in a short period of time and are great for reference anytime.

<a name="formatting-code"></a>
### üíª Formatting Code

Whenever you mention a file name like `App.jsx`, a variable reference like `movie.title`, etc., you should use *inline styling* with **single backticks** enclosing the text to highlight.

If you have a *entire block of code*, or perhaps *one or more commands at a prompt*, surround it with **triple backticks** instead. Examples:

```shell
cd project-directory
npm run dev
```

```javascript
function sayHello(name) {
    console.log(`Hello, ${name}!`);
}

sayHello("Bob"); // Hello, Bob!
```

<a name="emoji"></a>
### üòÑ Emoji
You can copy-and-paste emoji into your `.md` document, or if you are editing on GitHub, you can use `:emojicode:` (similar to Slack) with the tab key to insert emoji as you go.

> [!TIP]
> For portfolio project READMEs, use emoji sparingly.
